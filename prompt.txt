我正在编写一个基于electron的AI自动阅卷app。

我现在告诉你我整个应用的规划，以及预计的用户使用这个应用的工作流，你先帮我规划一下整个项目的结构，以及预计需要用到的依赖等。

然后完成初步的布局，可以暂时不用接入实际功能。我希望使用 electron 架构，使用 vue + vite来构建整个项目的前端，然后选择合适的现代UI框架来完成整个项目构建。我希望最终的程序能运行在 windows + macos + ubuntu 上，现在我的开发环境是macOS，程序的包名最终可以是 "cn.bit-helper.neuromark"。

我在项目目录放了一个 reference_python_project 是我之前用于原型验证编写的python工作流代码，在接下来的要求中，我会要求你有的时候参考一下里面的实现代码。

我先给你介绍整个工作流吧：

- 我希望用户使用这个软件进行AI阅卷是项目制的，比如用户想要批阅一套新的卷子，那么就是新建一个项目，然后这个项目里面可以包含很多的学生试卷，用户可以以上传图片的方式上传学生的试卷（一个学生可能会有多张试卷图片），也可以通过我们这个APP的手机端，拍摄图片后通过局域网自动发送到桌面端（手机端我们可以设计一个拍照系统，你暂时不用考虑手机端任何事情）。所有这些上传的试卷图片就会形成一套原始答卷库，然后用户可以上传针对这个项目的参考答案文本（支持markdown格式，渲染的时候也需要支持渲染markdown以及内联latex公式），之后的阅卷过程，llm就会严格参考这一套的阅卷评分标准进行评分。

- 在阅卷之前，用户需要进行批量“试卷扫描识别”工作，这个过程你可以参考代码：reference_python_project/batch_scan_documents.py，实际上就是将“原始答卷库”中的所有照片拿出来识别A4纸边界，然后转成扫描件的样式方便后续AI进行阅卷，这个过程形成的扫描档会存储在项目的“扫描答卷库”。用户可以在app可视化查看扫描的情况，包括纸张边界标注情况，以及实际扫描的结果等等。

- 用户可以在APP全局设置里面配置用于阅卷的多模态llm后端，包括设置api key，我们都采用openai的调用标准接口就行，设置界面设置后端要支持测试是否可用。还可以设置批阅的并行数，默认先设置为1，便于调试。

- 然后一切准备就绪过后，就可以参考 reference_python_project/batch_grade_documents.py 的代码开始对项目内的“扫描答卷库”进行批量批阅了，要求能实时查看批阅进度，显示批阅速度，支持随时打断，重新批阅时支持跳过已经批阅的答卷，支持按照 “全局设置” 中的批阅并行数进行批阅。

- 在批阅的过程中，用户可以随时切换到已经批阅的结果列表查看已经批阅完成的答卷的评分结果（此时保证并行批阅可以依然在后台运行），具体的查看信息需要支持markdown格式的渲染，你可以参考reference_python_project/student_result_lookup.html 和 reference_python_project/result_viewer.html两个页面的渲染设计。用户还可以在结果页面修改被模型错误识别的学号，姓名等信息，然后进行保存。用户可以分别对每个小题的分数进行修改，或者直接修改总分，修改后需要保存修改的结果。

- 当项目批阅完成过后，用户可以在查看结果的页面选择导出json格式的评阅结果。

- 额外的一个点，我希望在开始批阅时，用户的界面可以添加一个开关叫“绘制批阅区域”的选项，默认是关闭的（因为考虑到有的模型并不支持这个功能），如果打开，那么在批阅的时候需要额外给llm后端注入一段提示词，让模型在批阅的时候输出每个小题的批阅区域坐标（可以返回多个）（为了适配多张图片，应该还需要模型返回是第几张图的，并且返回的应该是相对坐标0-1.0，以防止图片压缩导致坐标错误），然后我们在结果页面可以根据这些坐标在原始答卷图片上绘制出批阅区域的边界框（不同题目用不同颜色标注），方便用户查看模型是如何进行批阅的。

- 整体界面应该就是根据tab栏导航分为了几个部分：

-- 1. 项目页面：显示用户当前创建的所有的项目，然后有基本的信息，比如有多少套导入的试卷？多少已经扫描，多少已经批改？平均分等基本信息

-- 2. 后台任务页面：显示当前正在进行批改的后台任务，包括批改的进度，任务所属的项目名，批改的速度，预计的结束时间等等信息。

-- 3. 参考答案生成页面：这个页面就是干一件事，不跟app的其他页面联动或者有数据交互。功能就是内置了一些提示词，让用户可以选择，并上传试卷原始题目的图片，然后到时候就会调用后端的LLM模型去生成markdown格式的参考答案，然后生成完毕后可以给用户展示，供用户进行修改，试试渲染markdown格式的效果，并支持用户复制操作。另外也需要保存所有生成的参考答案的历史，形成列表。

-- 4. 全局设置页面：目前设置页面可以暂时只让用户设置llm api后端的相关信息，并行数、是否显示标注边界等与项目挂钩的设置在项目设置里面去做。
